Statistics 설명

Number of hash/redux: Ophcrack이 시도한 해시 및 리덕션(reduction) 작업의 총 횟수로, 해시 값과 비밀번호의 추정 값을 일치시키기 위해 여러 번 시도한 횟수다.
Number of fseek: 파일 내에서 fseek 함수를 호출한 횟수로, 특정 위치를 검색할 때 사용되지만, 이 실험에서는 0으로 표시된다.
Number of false alarms: 비밀번호 추정 과정에서 잘못된 일치를 발견한 횟수다. 레인보우 테이블을 사용할 때 가끔 잘못된 매칭이 발생할 수 있다.
Num. hash/redux per false alarm: 잘못된 매칭이 발생하기까지의 평균 해시 및 리덕션 횟수다.
Num. of matches in tables: 레인보우 테이블에서 발견된 비밀번호의 총 개수로, 테이블을 이용하여 추출한 비밀번호 수를 나타낸다.
Num. of matches in brute force: 무작위 대입(Brute Force) 방식으로 찾은 비밀번호의 총 개수로, 레인보우 테이블에 없는 비밀번호를 무작위로 시도하여 찾는 방법을 사용한 결과다.

오른쪽 Details 패널:

Number of prefixes/postfixes found: 레인보우 테이블을 통한 매칭 과정에서 발견된 접두사/접미사의 수를 나타낸다.
Number of start found: 매칭에 사용된 시작 포인트 수다.
Number of fseek in .index/.bin/.start: 각 파일에 대한 fseek 호출 횟수로, 이 실험에서는 모두 0으로 표시된다.

암호 해독 메커니즘 설명

이 실험은 레인보우 테이블을 사용하여 해시 값을 통해 비밀번호를 복구하는 방식이다. Windows의 계정 비밀번호는 해시 값으로 저장되며, 이를 일반적으로 복구하려면 매우 많은 계산이 필요하다. 그러나 레인보우 테이블은 미리 계산된 해시 값과 비밀번호 쌍을 포함하여 이 작업을 가속화할 수 있다.

레인보우 테이블 탐색:

Ophcrack은 미리 계산된 레인보우 테이블에서 대상 해시와 일치하는 해시 값을 찾는다. 테이블에서 일치하는 해시가 발견되면 해당 해시에 대응하는 비밀번호를 복구할 수 있다.
False Alarms 처리:

레인보우 테이블은 잘못된 매칭을 일으킬 수 있어, 프로그램은 이를 검증하고 다시 시도하는 과정을 거친다. 통계에 나타난 "false alarms"가 이와 관련된 수치다.
Brute Force:

테이블에서 발견되지 않는 해시는 무작위 대입 방식으로 복구를 시도하며, 이는 테이블 방식보다 느리지만 레인보우 테이블에 없는 비밀번호를 찾을 수 있다.
이 실험에서는 레인보우 테이블과 무작위 대입 방식을 조합하여 다양한 해시 값을 비밀번호로 복구해 내는 과정을 수행하였다.


이번 실험은 Ophcrack이라는 프로그램을 이용하여 Windows 운영 체제에서 사용되는 해시된 비밀번호를 복구하는 과정을 수행하였다. 이 과정은 해시된 비밀번호를 일반 텍스트로 변환하는데 주로 레인보우 테이블이라는 미리 계산된 데이터를 사용하여 복구 시간을 단축하는 기법을 사용하였다.
